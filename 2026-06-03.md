## I. TỔNG QUAN

VIETSHOP là web app quản lý nhập hàng, tồn kho, bán hàng, và thống kê lợi nhuận. Phục vụ cho nhóm kinh doanh hàng hóa (đồ chơi, mô hình, đồ sưu tầm,...).

**Kiến trúc:** API backend (Django DRF) + Frontend (Next.js SSR) + Nginx reverse proxy.
**Chạy bằng Docker Compose** — 4 containers.

### Công nghệ

| Layer | Công nghệ |
|-------|-----------|
| Backend | Python 3.12, Django 5, DRF, PostgreSQL 16 |
| Frontend | Next.js 16 (App Router), TypeScript, Tailwind v4, Shadcn UI, Recharts |
| Auth | JWT (SimpleJWT) — access token 30p + refresh token 1 ngày |
| Images | Cloudinary (upload + crop + resize) |
| Infra | Docker Compose, Nginx |
| Payment | **Chưa có** — cart/checkout chưa làm |

## III. CÁCH CHẠY

### Docker (recommended)

```bash
# Build và start toàn bộ
docker compose up --build -d

# Chỉ rebuild frontend (nhanh hơn khi sửa frontend)
docker compose up --build -d nextjs

# Xem log
docker compose logs -f

# Dừng
docker compose down

# Reset toàn bộ DB (xóa volumes)
docker compose down -v && docker compose up --build -d
```

Sau khi chạy, truy cập: **http://localhost**

### Development (không Docker)

**Backend:**
```bash
cd backend
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env   # điền thông tin
python manage.py migrate
python scripts/seed.py && python scripts/seed_demo.py
python manage.py runserver 0.0.0.0:8000
```

**Frontend:**
```bash
cd frontend
npm install
NEXT_PUBLIC_API_URL=http://localhost:8000/api npm run dev
```

⚠️ Cần PostgreSQL hoặc sửa settings để dùng SQLite.

## IV. CÁC MODULE CHÍNH

### 1. Dashboard (Admin)
- Backend API: `backend/apps/dashboard/views.py` — 5 endpoints
- Frontend: `frontend/src/app/admin/dashboard/page.tsx`
- 4 stat cards + 4 biểu đồ (Revenue, Profit, Inventory, TopCategories) + 3 monthly cards
- **Không còn** thống kê nhập/xuất (đã xóa 03/06)

### 2. Products
- CRUD đầy đủ (có edit mode) + filter (status, category, date range, keyword)
- Image upload lên Cloudinary (crop trước khi upload trong Admin)
- Status workflow: `in_stock` → `sold` / `pending_price` / `cancelled`
- Product view tracking: chỉ đếm anonymous + customer, skip admin/staff

### 3. Purchases (Nhập hàng)
- Tạo phiếu nhập → auto cập nhật `purchase_price` của sản phẩm
- Có validation: không cho nhập sản phẩm đã `sold` hoặc đang `in_stock`
- Có datetime-local picker (default = now)
- **Thiếu:** edit UI (sửa phiếu nhập)

### 4. Sales (Bán hàng)
- Tạo phiếu bán → auto set `status=SOLD`, `sale_price`
- Có `select_for_update()` tránh race condition
- Chỉ bán được sản phẩm chưa `SOLD`
- Có datetime-local picker (default = now)
- **Thiếu:** edit UI (sửa phiếu bán)

### 5. Blog
- CRUD với danh mục, featured image (Cloudinary), draft/publish
- Admin có edit mode
- Blog category có đầy đủ CRUD (list, create, update, delete)
- **Thiếu:** public detail page (hiện chỉ có list)

### 5a. Account
- Trang `/account` cho user đã đăng nhập
- Xem thông tin cá nhân

### 6. Audit Logs
- Tự động ghi lại mọi thay đổi trong hệ thống
- Dùng thread-local middleware để track user (`core/audit_middleware.py`)
- API endpoint: `/api/audit-logs/`

### 7. Settings (Singleton)
- Footer settings (company, address, social links)
- Chỉ được 1 record — `has_add_permission` trả về False

### 8. Slider
- Banner hero slider, Admin CRUD, dùng ở Home page
- Upload ảnh + link đích, sắp xếp thứ tự, bật/tắt

### 9. Notifications
- Banner thông báo có thời hạn hiệu lực, hiển thị trên public
- Dùng ở layout chung

## VIII. LƯU Ý CHO NGƯỜI MỚI

1. **Next.js 16 có breaking changes** — đọc `frontend/AGENTS.md` và docs trong `node_modules/next/dist/docs/` trước khi viết code
2. **Template first** — Luôn đối chiếu `frontend/public/template/*.html` trước khi sửa public page
3. **Theme tokens** trong `frontend/src/app/globals.css` — dùng `@theme` tokens, không hardcode
4. **JWT client-side** — Token trong localStorage, Axios interceptor tự động refresh
5. **Không redirect 401** — Public pages không redirect `/login` khi token hết hạn
6. **API proxy** — Tất cả API gọi qua `/api/*` (Nginx), không gọi trực tiếp Django
7. **Không có unit test** — Backend + Frontend đều không có test
8. **Seed data chạy tự động** khi docker start, nếu sửa model thì xóa volume: `docker compose down -v && docker compose up --build -d`
9. **Cloudinary** là nơi lưu ảnh — cần tài khoản free và điền `.env`
10. **File ghi chép công việc** theo format `DD-MM-YYYY.md` — xem `02-06-2026.md` làm mẫu
