## 2. TopCategoriesChart — Hết khoảng trống, auto-height

**Vấn đề**: Biểu đồ thanh bị thừa khoảng trống cả chiều ngang lẫn chiều dọc. X-axis hiển thị 0→12 trong khi data chỉ max 3.

**Fix**:
- Đổi `h-[250px]` → dynamic height `Math.max(160, data.length * 25 + 40)` (tự động co giãn theo số danh mục)
- Thêm empty state `"Không có dữ liệu danh mục"` khi data rỗng
- Tính `maxCount` bên ngoài JSX: `Math.max(...data.map(d => d.count), 1)`
- XAxis domain: `[0, Math.ceil(maxCount * 1.2)]` — trục hoành chỉ tới ~4 thay vì 12, thanh bar dài full width

**File**: `frontend/src/components/charts/TopCategoriesChart.tsx`

## 3. Admin Layout — Menu biến mất khi scroll

**Vấn đề**: Sidebar dùng `lg:sticky lg:top-0 lg:h-screen`, nhưng container cha `min-h-screen` phình to hơn viewport → sticky "hết range" → sidebar bị đẩy lên trên, mất tiêu khỏi màn hình.

**Fix**:
- Container cha: `min-h-screen` → `h-screen overflow-hidden` (cố định = 100vh)
- Content column (phải): thêm `overflow-y-auto` (chỉ content cuộn độc lập)
- Sidebar sticky giờ hoạt động đúng vì container cha = viewport height

**File**: `frontend/src/app/admin/layout.tsx`

## 4. NaN trên Purchases/Sales — Tổng chi phí, Doanh thu

**Vấn đề**: `Tổng chi phí` (Purchases) và `Doanh thu` / `TB mỗi đơn` (Sales) hiển thị `NaN ₫`.

**Root cause**: Django DRF serialize `DecimalField` thành string (`"150000.00"`) trong JSON. Frontend dùng:
```typescript
purchases.reduce((sum, p) => sum + p.purchase_price, 0)
// 0 + "150000.00" → string concatenation "0150000.00"
// "0150000.00" + "200000.00" → "0150000.00200000.00" (2 dấu chấm) → NaN
```

**Fix**: Thêm `Number()` vào reduce:
- `purchases/page.tsx`: `sum + Number(p.purchase_price)`
- `sales/page.tsx`: `sum + Number(s.sale_price)`

**Files**:
- `frontend/src/app/admin/purchases/page.tsx`
- `frontend/src/app/admin/sales/page.tsx`
