### Phase 5: Fix lỗi sau deploy

**Triệu chứng:** API "đang tải", "không tìm thấy sản phẩm", giao diện mobile bị bự, blog không load.

**Nguyên nhân gốc rễ:** Trong `docker-compose.yml`:
```yaml
NEXT_PUBLIC_API_URL: http://localhost/api   # ← SAI!
```
Giá trị này được baked vào JS khi build Docker. Trình duyệt người dùng gọi API đến `http://localhost/api` (máy họ) thay vì domain thật → tất cả API đều chết.

**Các fix đã thực hiện và push lên GitHub (commit `dc8435b`):**

| # | File | Lỗi | Fix |
|---|------|-----|-----|
| 1 | `docker-compose.yml` | `NEXT_PUBLIC_API_URL: http://localhost/api` | Đổi thành `/api` (đường dẫn tương đối) |
| 2 | `frontend/src/app/layout.tsx` | Thiếu viewport meta → mobile bị bự | Thêm `export const viewport: Viewport = { width: "device-width", initialScale: 1 }` + đổi lang `en` → `vi`, update title/description |
